# PROJECT ARCHITECTURE: Meu Projeto

## 1. CONTEXT & PROBLEM

Pessoas físicas têm dificuldade em controlar suas finanças pessoais de forma simples e visual. As informações ficam dispersas entre extratos bancários, planilhas manuais e anotações avulsas. Não há uma visão consolidada de receitas, despesas e saldo, o que leva a descontrole financeiro, gastos desnecessários e falta de planejamento.

## 2. PROPOSED SOLUTION

Um web app de gestão financeira pessoal que permite ao usuário registrar receitas e despesas, categorizá-las, e visualizar um dashboard com resumo mensal (total de receitas, total de despesas e saldo). A aplicação será responsiva, com autenticação via Supabase Auth e dados persistidos no Supabase (PostgreSQL).
DIFERENCIAIS
Interface moderna e minimalista com shadcn/ui, dashboard visual com gráficos de categorias, filtros por período e categoria, e deploy instantâneo na Vercel. Todo o desenvolvimento será feito com Claude Code como agente de codificação.


## 3. FUNCTIONAL REQUIREMENTS

- Login e Autenticação
- Dashboards
- Busca e Filtros

•	Autenticação de usuários via Supabase Auth (email/senha)
•	Cadastro de transações (receitas e despesas) com: valor, data, categoria, descrição
•	Listagem de transações com filtro por período e categoria
•	Dashboard com resumo mensal: total de receitas, despesas e saldo
•	Gráfico de despesas por categoria (Recharts)
•	Editar e excluir transações
•	Responsivo (mobile-first)

## 4. USER PERSONAS

Pessoa física que deseja organizar suas finanças. Cadastra transações, acompanha o dashboard e exporta relatórios.

Backend/BaaS: Supabase (PostgreSQL + Auth + Row Level Security). Sem backend separado.
Deploy: Vercel (integração contínua via GitHub).
Desenvolvimento: Claude Code como agente de codificação. Versionamento com Git/GitHub.
